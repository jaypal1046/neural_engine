# Algorithmic Art Skill

Create original generative art in two phases: **Philosophy → Code**.
Never copy existing artists' styles directly. Create original computational aesthetics.

---

## Phase 1: Algorithmic Philosophy

Before writing any code, write a generative philosophy — a manifesto for a new computational
aesthetic movement that will guide the implementation.

### What the Philosophy Is

A 4–6 paragraph document describing:
- The **movement's name** (1–2 words, e.g. "Organic Turbulence", "Stochastic Crystallization")
- The **computational worldview** — what mathematical or natural processes embody it?
- How it manifests through: noise functions, particle behaviors, field dynamics, emergence, time
- The **tone of craftsmanship** — stress that the algorithm should feel like it took deep expertise
  and many iterations to arrive at. Use phrases like "meticulously tuned", "refined through
  countless iterations", "the mark of deep computational mastery" — but only once each, not repeated.

### Philosophy Examples (condensed — expand to 4–6 paragraphs in practice)

**"Organic Turbulence"** — Chaos constrained by natural law. Flow fields from layered Perlin noise.
Thousands of particles following vector forces, trails accumulating into organic density maps.
Color emerges from velocity — fast particles burn bright, slow ones fade. Every parameter tuned
to the balance point between chaos and calm by someone at the top of their field.

**"Quantum Harmonics"** — Discrete entities exhibiting wave interference. Particles on a grid,
each carrying phase values that evolve through sine waves. Constructive interference creates
bright nodes, destructive creates voids. Simple harmonic motion → complex emergent mandalas.

**"Stochastic Crystallization"** — Random processes crystallizing into order. Voronoi tessellation
via random point relaxation. Cells push apart to equilibrium. Color from cell size and neighbors.
The organic tiling feels both random and inevitable — master-level generative algorithm.

### Conceptual Seed (Important)

Before coding, identify a **subtle conceptual thread** from the user's request to embed in the
algorithm. Not literal — sophisticated. Like a jazz musician quoting another song: only those
who know will catch it, everyone else just experiences beautiful art. Weave it into parameters,
behaviors, color choices.

**Output**: Save the philosophy as a `.md` file.

## Phase 2: p5.js Implementation

### Step 0: Read the Template (Required)

Read `templates/viewer.html` using the view tool before writing any HTML.
Use it as the literal starting point — not inspiration. Copy its structure exactly.

If `templates/viewer.html` is not found: use the **Fallback Structure** in `references/fallback.md`.

### What's Fixed vs Variable in the Template

**FIXED — copy exactly, never change:**
- Overall layout (header, sidebar, main canvas area)
- Anthropic branding (Poppins/Lora fonts, light colors, gradient backdrop)
- Seed section: seed display, Prev/Next/Random buttons, Jump-to-seed input
- Actions section: Regenerate, Reset, Download PNG buttons

**VARIABLE — create fresh for each artwork:**
- The entire p5.js algorithm (`setup()`, `draw()`, all classes)
- The `params` object (what this specific art needs to be tunable)
- The Parameters sidebar section (sliders/inputs matching the params)
- The Colors section (include only if the art needs adjustable palettes)

Every artwork must have a **unique algorithm and unique parameters**. The fixed shell provides
consistent UX — the variable parts express the unique vision.

### Algorithm Design Principles

Let the philosophy guide what to build. Ask: *"How does this philosophy express itself in code?"*
not *"Which pattern should I use?"*

- **Emergence philosophy** → elements that accumulate, grow, interact over time
- **Mathematical beauty** → geometric relationships, trigonometric harmonics, precise calculations
- **Controlled chaos** → random variation within strict boundaries, order from disorder
- **Field dynamics** → vector fields, forces acting on particles, invisible made visible

See `references/generator_template.js` for p5.js best practices (code structure, seeded randomness,
class patterns, utility functions). Read it when you need p5.js structural guidance.

### Required Technical Patterns

**Seeded randomness — always:**
```javascript
let params = { seed: 12345, /* ...art-specific params */ };

function initSeed(seed) {
    randomSeed(seed);
    noiseSeed(seed);
}
```
Same seed must always produce identical output. This is non-negotiable.

**Parameter object:** One object for all tunable values. Connect every param to a UI control.

**Canvas:** 1200×1200px standard. p5.js from CDN:
`https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js`

**Self-contained:** Single HTML file. No external files except p5.js CDN. Everything inline.

### Craftsmanship Standard

The algorithm must feel like it emerged from deep expertise and careful iteration — not default
noise with random colors. This means:
- Color palettes that are intentional and harmonious
- Complexity that rewards close inspection
- Parameters tuned so defaults look immediately beautiful
- Composition that has visual hierarchy even in randomness

### Output

1. **Philosophy** — `.md` file
2. **Interactive HTML artifact** — single self-contained file built from the template:
   - p5.js algorithm expressing the philosophy
   - Working seed navigation (prev/next/random/jump)
   - Parameter controls (sliders for every tunable value)
   - Regenerate, Reset, Download buttons
   - Anthropic branding preserved
