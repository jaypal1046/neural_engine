Question: What is Huffman coding?

Ideal Answer: Huffman coding assigns shorter bit codes to more frequent symbols and longer codes to rare ones, achieving optimal prefix-free encoding.

**Steps:**
1. Count frequency of each symbol (byte value 0–255)
2. Build a binary tree: repeatedly merge the two lowest-frequency nodes
3. Assign bit codes: left branch = '0', right branch = '1'
4. Most frequent symbol gets the shortest code

**Example:** If 'e' appears 40% of the time, it might get code `10` (2 bits) instead of the standard 8 bits — a 75% saving for that symbol.

Our engine uses canonical Huffman coding with length-limited codes (max 15 bits) in the LZ77 and BWT pipelines.
