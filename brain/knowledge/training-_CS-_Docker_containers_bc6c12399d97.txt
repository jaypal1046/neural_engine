Question: What is Docker?

Ideal Answer: Docker packages an application and all its dependencies into a **container** — a lightweight, portable, isolated environment that runs consistently on any machine.

**Key concepts:**
- **Image:** blueprint for a container (like a class)
- **Container:** running instance of an image (like an object)
- **Dockerfile:** script that builds an image
- **Registry:** store of images (Docker Hub)

**Why Docker?** Solves 'it works on my machine' — the container includes OS libraries, config, everything.

**Example Dockerfile:**
```dockerfile
FROM python:3.11
COPY app.py .
RUN pip install flask
CMD ["python", "app.py"]
```

Build: `docker build -t myapp .`
Run: `docker run -p 8080:8080 myapp`
