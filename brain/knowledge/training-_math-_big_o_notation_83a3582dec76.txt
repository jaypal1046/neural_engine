Question: What is Big O notation?

Ideal Answer: Big O notation describes how an algorithm's time or space complexity grows as input size n increases.

**Common complexities (best to worst):**
- O(1) — Constant: array index lookup
- O(log n) — Logarithmic: binary search
- O(n) — Linear: single loop through array
- O(n log n) — Quicksort, mergesort
- O(n²) — Quadratic: nested loops (bubble sort)
- O(2ⁿ) — Exponential: brute-force combinations

**Rule:** Drop constants and lower-order terms. O(3n + 50) → O(n).
