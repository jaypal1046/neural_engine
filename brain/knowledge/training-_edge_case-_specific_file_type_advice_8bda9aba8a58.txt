Question: I have a JSON file with 50,000 records. Which algorithm should I use?

Ideal Answer: Great question — for JSON specifically:

1. **Run analyze first:** `analyze yourfile.json`
   JSON has high structural regularity (lots of `:{`, `","`, brackets) → typically 4.5–6.5 bits/byte entropy.

2. **Recommended:** `--best` (BWT+MTF+ANS) — JSON's repeated key names are perfect for BWT clustering. Expect 60–70% compression.

3. **For maximum ratio:** `--cmix` — the 1,046 advisors learn JSON patterns deeply. Slower but can reach 75–80% compression.

4. **Avoid:** Default LZ77 alone — it works but wastes the structural patterns JSON offers.

For 50,000 records, I'd benchmark `--best` vs `--cmix` and pick what fits your speed requirement.
