  Command Implemented:
    ✅ neural_engine train_transformer_mixed <corpus> [options]

  Supported Modes:
    ✅ FP16 (IEEE 754 half-precision)
       - 16-bit floating point
       - Range: ±65504, Precision: ~3 decimal digits
       - Dynamic loss scaling enabled automatically
       - 2-3x faster training (expected)
       - 50% less memory

    ✅ BF16 (bfloat16)
       - 16-bit brain floating point
       - Same range as FP32, less precision
       - NO loss scaling needed (easier to use)
       - 1.5-2x faster training (expected)
       - 50% less memory

    ✅ FP32 (full precision baseline)
       - 32-bit floating point (default)
       - Maximum precision and stability
       - Baseline for comparison

  Command Options:
    --mode <FP16|BF16|FP32>    Precision mode (default: FP16)
    --epochs <N>               Number of epochs (default: 7)
    --lr <float>               Learning rate (default: 0.002)
    --batch <N>                Batch size (default: 16)
    --loss-scale               Enable dynamic loss scaling (auto for FP16)

  Key Features Implemented:
    ✅ FP16/BF16 conversion functions
       - fp32_to_fp16() / fp16_to_fp32()
       - fp32_to_bf16() / bf16_to_fp32()

    ✅ Dynamic Loss Scaler
       - Scales loss to prevent gradient underflow
       - Automatically adjusts scale based on overflows
       - Default scale: 65536 (2^16)
       - Scale window: 2000 steps before increase

    ✅ Mixed Precision Optimizer
       - Stores master weights in FP32 (high precision)
       - Performs forward/backward in FP16/BF16 (fast)
       - Updates master weights (stability)

    ✅ Training Loop
       - BPE tokenizer integration
       - Batch creation from corpus
       - Loss tracking per epoch
       - Overflow detection and reporting
