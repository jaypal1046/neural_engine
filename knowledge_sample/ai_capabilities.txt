## Compression Capabilities

### Multi-Algorithm Compression
- Default (LZ77): Fast, good for binary data
- Best (BWT): 90-99% compression for text, slower
- Ultra (PPM): Maximum compression, very slow
- CMIX (Neural): Neural network compression
- Multi-threaded: 8 cores, 4-8x faster
- SIMD-optimized: SSE2/AVX2, 18% faster decompression

Commands:
- compress <input> [--best | --ultra | --cmix]
- decompress <input>

API:
- POST /api/compress - Compress file
- POST /api/decompress - Decompress file
- POST /api/compress_stream - Streaming compression
- POST /api/decompress_stream - Streaming decompression

### Block-Based Random Access
- Decompress specific blocks without reading entire file
- 9.20 MB/s block access speed
- 16 MB blocks for memory efficiency
- Perfect for large knowledge modules

Performance Metrics:
- Compression Ratio: 85-99% saved
- Compression Speed: 4-8x faster (8 threads)
- Decompression Speed: 7.11 MB/s (SIMD)
- Block Access: 9.20 MB/s
- Memory Usage: 16 MB/block
